Ordered list of nice-to-have features:

  1. should find a way to inspect a Relationship and view arrows to the parent dependencies (predecessors)... and also show all the relationships that depend on it (successors)... Best if those dependencies also track and indicate the type of Logic Element used to derive it (perhaps with a colour code, when presented visually)
  2. hints -> application provides hints when Automatic Processing is not enabled
  3. RollBack and Undo for constraints and clues
  4. when loading a problem from file, it would be interesting to have the option to step through processing of clues, or have it automatically process everything (Preferences)
  5. Android version (see http://ladstatt.blogspot.ca/2016/02/fx-tictactoe-deploy-javafx-app-into.html)
  6. internationalization support, particularly for UI and Ordinals
  7. automatic discovery of SpecialCategory types based on list of known words, but users will be able to force ordinals if words are not known from a list. The list should be in a text file with headings such as [weekdays] that match the category types and a different text file can be generated for each language
  8. improved validation on unmarshalling XML files
  9. FXLauncher for automatic app update https://github.com/edvin/fxlauncher
  10. aim to automatically recognize special categories automatically when entering items, and prompt user to validate
  11. auto-complete when entering clues, based on recognized items or category names (see TextField from ControlsFX)

Completed features

  1. notification pane for errors
  2. reset problem where the clues and constraints are removed but categories and items remain, akin to starting the problem over
  3. print API to print the grid with or without the Relationship, with or without labels
  4. provide a .lps file format used to rapidly input problems
  5. set/change category or item labels from the Grid using Lightweight dialogs
  6. provide zoom in/zoom out capability (see example in Mastering... p.48) using buttons on the top left of the Grid (see Layout)
  7. mnemonic and shortcut keys for menu items
  8. provide tool tips
  9. package some sample problem files with the .jar file and extract them from the .jar file when the application is launched

Abandonned features

  1. add look and feel options or skins.... Look at p.159- for look and feel setting code.
